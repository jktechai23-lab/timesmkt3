IMKT3 — UI read-only

Painel de inspeção do timesmkt3 (bot + orchestrator + workers). Read-only — a UI não dispara renders nem modifica nada no disco.

Os arquivos são lidos diretamente das pastas: prj/<projeto>/outputs/<campanha>_<data>/ contém ads, imgs, video, platforms e logs por campanha.

Aba Campanhas

Cada card é uma campanha gerada pelo bot/orchestrator. Mostra contadores de ads (PNG), imgs (API) e vídeos (mp4).

  • Filtro de projeto (topo) limita às campanhas de um cliente específico.
  • Busca filtra por nome, task_name, data ou projeto.
  • Cada campanha vive em prj/<projeto>/outputs/<task_name>_<date>/.
  • Clique nos botões ads, imgs, vídeos de cada card pra abrir lightbox/modal.
Aba Config

Mostra metadata do projeto: versão (do package.json), Node, root path. Útil pra confirmar em qual repo/build a UI está apontando.

Vídeos do Estúdio (mkVIDEOS)

Os vídeos autorais (criaprof, gertran, darkstory, storytree, etc) não vivem mais aqui — foram migrados pro projeto separado ~/projetos/mkvideos/.

  • UI dos vídeos: http://127.0.0.1:5278 (mkVIDEOS · porta 5278)
  • Pra gerar novo vídeo, rode node gen-<tipo>.js no repo do mkvideos.
CLI relacionados (ainda no timesmkt3)
  • npm run bot — bot do Telegram (orchestrator de campanhas)
  • npm run pipeline:worker — worker BullMQ que processa jobs
  • npm run pipeline:run — dispara um payload demo
  • npm run pipeline:run:payload <json> — dispara payload inline
  • npm run campaign-import:scan — importa campanhas externas
  • npm run media:status — status dos provedores de mídia
  • npm run check:syntax — checa sintaxe de todos os JS
  • npm run ui — esta UI (porta 5178)
